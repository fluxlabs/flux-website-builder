# Requirements: Flux Website Builder

## 1. Setup (SETUP)
- **SETUP-01:** Initialize Next.js project with TypeScript and Tailwind CSS.
- **SETUP-02:** Configure environment variables (Vercel, GitHub, OpenAI, etc.).

## 2. Marketing (MKT)
- **MKT-01:** Professional landing page with 48-72 hour turnaround messaging.
- **MKT-02:** Service tiers and pricing models display.
- **MKT-03:** Portfolio and testimonials sections (placeholders).
- **MKT-04:** Primary CTA leading to the intake form.

## 3. Client Intake (INTAKE)
- **INTAKE-01:** Multi-step intake form collecting basic info (Name, Email, Business Name, URL).
- **INTAKE-02:** Design preferences collection (Inspiration links, Color palette, Logo upload).
- **INTAKE-03:** Content & Goals collection (Site goal, Required pages).
- **INTAKE-04:** Submission summary and confirmation message with 48-72 hour countdown.
- **INTAKE-05:** Store intake data in a database (Supabase/Postgres).

## 4. AI Generation Pipeline (AI)
- **AI-01:** AI Agent to ingest intake data and generate Next.js source code.
- **AI-02:** Component-based prompting strategy for reliable code generation.
- **AI-03:** Local validation/compilation step for AI-generated JSX/TSX.
- **AI-04:** Structured JSON schema for AI output.

## 5. Automation (AUTO)
- **AUTO-01:** Programmatic creation of a GitHub repository for each client.
- **AUTO-02:** Automatic push of AI-generated code to the client's GitHub repo.
- **AUTO-03:** Vercel project creation and linkage via Vercel API.
- **AUTO-04:** Deployment to a staging URL and notification to the client.

## 6. Approval & Launch (HAND)
- **HAND-01:** Simple internal dashboard to track build statuses.
- **HAND-02:** Client approval workflow for the staging site.
- **HAND-03:** Interactive "Launch Guide" (Web or PDF) with DNS instructions.
- **HAND-04:** Automated SSL provisioning via Vercel.
- **HAND-05:** DNS/Domain pointing assistance (A records, CNAMEs).
